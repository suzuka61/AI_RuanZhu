# 秒著 - AI软著材料生成系统 PRD

> 版本：v1.7
> 更新日期：2026-03-26
> 状态：已完成

---

## 1. 产品概述

### 1.1 产品定位
「秒著」是一款基于AI的软件著作权申请材料自动生成工具。用户上传项目文档（.doc/.docx/.pdf），系统通过AI提取关键信息，自动生成符合国家版权局要求的「软件概述」和「设计说明书」两份核心申请材料，并导出为标准格式的Word文档。

### 1.2 目标用户
- 需要申请软著的个人开发者
- 中小企业技术负责人
- 知识产权代理机构

### 1.3 核心价值
- 将传统需要1-3天的软著材料撰写工作缩短至几分钟
- 降低软著申请的专业门槛
- 生成符合官方规范的标准化文档

## 2. 系统架构

### 2.1 技术栈
| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | 单HTML文件（原生JS） | 零依赖，浏览器直接运行 |
| 后端代理 | Node.js (proxy.js) | 解决跨域 + 生成docx |
| AI服务 | 火山引擎API | 支持多模型切换 |
| 文档解析 | mammoth.js / pdf.js | 前端解析docx和pdf |
| 文档生成 | docx (npm) | 服务端生成Word文档 |
| 图片处理 | JSZip | 从docx中提取嵌入图片 |

### 2.2 部署架构
```
浏览器 (秒著.html)
    ↓ HTTP
Node.js 代理 (localhost:3000)
    ├── /proxy → 火山引擎API (ark.cn-beijing.volces.com)
    ├── /generate-docx → 本地生成Word文档
    └── GET / → 提供HTML页面
```

### 2.3 支持的API提供商
| 提供商 | 标识 | 默认模型 | 免费 |
|--------|------|----------|------|
| OpenRouter | openrouter | stepfun/step-3.5-flash:free | ✅（需用户注册） |
| 火山引擎方舟 | volcengine | doubao-seed-2-0-code | ❌ |
| DeepSeek | deepseek | deepseek-chat | ❌ |
| 智谱 GLM | zhipu | glm-4.7-thinking | ❌ |
| 月之暗面 Kimi | moonshot | moonshot-v1-8k | ❌ |
| MiniMax | minimax | MiniMax-Text-01 | ❌ |

> **说明**：
> - v1.6版本新增OpenRouter免费模型，内置API Key，无需配置即可使用
> - v1.4版本起简化模型选择，用户只需选择API提供商，系统自动使用对应提供商的默认模型
> - 用户只要有API密钥，即可使用对应服务

## 3. 功能模块

### 3.1 Step 1 - 配置API
- 选择API提供商（下拉菜单：Step 3.5 Flash免费、火山引擎、DeepSeek、智谱、Kimi、MiniMax）
- 输入对应提供商的API Key（密码输入框，本地localStorage持久化）
  - Step 3.5 Flash免费模型：需输入在OpenRouter注册的API Key
- 系统自动使用提供商的默认模型，无需手动选择
- 保存后自动跳转下一步
- API Key和提供商选择持久化存储，下次打开自动恢复
- **布局优化**：API提供商、API Key、测试连接按钮全部放在同一行显示，减少空白
- **测试连接按钮**：移至API Key输入框右侧，操作更便捷

### 3.2 Step 2 - 上传文件
- 支持点击上传 / 拖拽上传
- 支持多文件上传
- 支持格式：.doc, .docx, .pdf
- 单文件大小限制：10MB
- 文件列表展示（文件名、大小、删除按钮）
- 备选：直接粘贴文档文本内容
- 文档解析：
  - docx：使用mammoth.js提取文本 + JSZip提取嵌入图片
  - pdf：使用pdf.js提取文本
  - 图片提取时记录所属章节标题（用于后续智能匹配）

### 3.3 Step 3 - 确认信息
- AI自动提取以下字段（调用LLM）：

**核心信息（默认展示）：**
| 字段 | 说明 | 是否必填 |
|------|------|----------|
| software_name | 软件全称（含版本号） | ✅ |
| short_name | 软件简称 | |
| version | 版本号 | |
| developer | 开发者/公司名称 | ✅ |
| completion_date | 开发完成日期 | |
| publish_date | 首次发表日期 | |
| purpose | 开发目的 | |
| main_features | 主要功能描述 | ✅ |

**高级参数（折叠展示，带自动填充默认值）：**
| 字段 | 默认值 |
|------|--------|
| publish_location | 中国 |
| programming_langs | Java |
| source_lines | 12000 |
| dev_os | Windows 11 64位 |
| dev_tools | IntelliJ IDEA |
| run_platform | Windows/Linux |
| runtime_env | JDK 17 |
| dev_hardware | Intel Core i7处理器，16GB内存，512GB SSD硬盘 |
| architecture | （AI提取） |
| industry | 通用 |
| team_members_raw | （仅从文档提取，不编造） |

- 所有字段可手动编辑修改
- 自动填充字段有绿色标记提示
- **布局优化**：采用Grid两列布局，普通字段一行显示两个，全宽字段独占一行，整体页面高度控制在1-2屏以内
- **响应式设计**：小屏幕自动切换为单列布局

### 3.4 Step 4 - 生成结果
- 分两步生成：
  1. **软件概述**（~3000 tokens）：包含软件基本信息表、开发目的、功能列表、技术特点
  2. **设计说明书**（~8000 tokens）：包含项目背景、目标、范围、时间表、需求描述及界面展示
- **流式输出**（v1.6新增）：
  - 实时预览生成内容，降低等待焦虑
  - 3行滚动显示，中间行清晰，上下行模糊效果
  - 实时显示字数统计
  - 预估剩余时间显示
- Tab切换预览两份文档的纯文本内容
- 复制到剪贴板功能
- 下载为.docx格式（调用后端 /generate-docx 接口）
- 重新开始按钮（重置所有状态）

## 4. 生成文档规范

### 4.1 软件概述.docx
- **封面**：软件名称 + "软件概述" + 公司名 + 日期
- **正文**：
  - 信息表（字段名-值 两列表格，17个字段）
  - 开发目的
  - 面向领域/行业
  - 软件的主要功能（序号-名称-描述 三列表格）
  - 软件的技术特点
  - 产品架构图（从上传文档提取的图片）
- **页眉**：软件名称 + 页码（第X页 共X页）
- **页脚**：无
- **表格样式**：标题行浅灰色背景（F2F2F2），内容行白色，深黑色边框（333333）

### 4.2 设计说明书.docx
- **封面**：软件名称 + "设计说明书" + 公司名 + 日期
- **目录页**：从AI生成内容中提取标题生成
- **正文章节**：
  1. 项目背景（200字+）
  2. 项目目标（功能架构图 + 功能目标表）
  3. 项目范围
  4. 项目时间表（人员分工表，仅使用真实提取的人员信息）
  5. 需求描述及界面展示（每个功能模块包含：User Story + 验收标准5条+ + [WIREFRAME]图片占位符）
- **图片嵌入**：
  - [ARCHITECTURE_IMAGE] → 架构图
  - [WIREFRAME] → 按章节标题相似度匹配上传文档中的截图
  - 未匹配图片放入附录
- **页眉**：软件名称 + 页码（第X页 共X页）
- **页脚**：无
- **表格样式**：支持Markdown表格自动转换为Word表格，标题行浅灰色背景（F2F2F2），内容行白色，深黑色边框（333333）

## 10. 用户操作流程

```
启动代理 (node proxy.js)
    ↓
打开浏览器 (localhost:3000)
    ↓
Step 1: 输入API Key → 选择模型 → 保存
    ↓
Step 2: 上传文档(.doc/.docx/.pdf) 或 粘贴文本
    ↓
    [前端解析文档 + 提取图片]
    ↓
Step 3: AI提取信息 → 用户核对/修改
    ↓
Step 4: AI生成两份文档 → 预览 → 下载.docx
```
