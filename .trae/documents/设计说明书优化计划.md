# 设计说明书生成优化计划

## 背景
当前 `秒著.html` 和 `proxy.js` 负责生成软著申请材料的设计说明书。根据用户反馈，需要进行以下优化：

1. **图表格式**：生成的图表信息不要使用 markdown 格式，要采用表格格式
2. **图片布局**：生成的图片和功能文字要放在一起，不要都堆积在末尾

---

## 当前问题分析

### 问题1：Markdown 表格格式

**当前代码位置**：`genDesignSpec` 函数（秒著.html 第 1849-1860 行）

当前生成的表格格式：
```markdown
| 序号 | 名称 | 功能 |
|------|------|------|
| 1 | 功能1 | 描述 |
```

这会在设计说明书中以纯文本 markdown 表格形式呈现，不够美观。

**目标**：改为类似以下的文本表格格式（使用 `contentToParasWithImages` 函数中的表格生成逻辑）：
```
┌────┬────────┬────────┐
│序号│  名称  │  功能  │
├────┼────────┼────────┤
│ 1  │功能1   │描述    │
└────┴────────┴────────┘
```

### 问题2：图片堆积在末尾

**当前代码位置**：`proxy.js` 第 460-527 行

当前逻辑：
1. `matchImagesToPlaceholders` 函数将 `[WIREFRAME]` 占位符与图片匹配
2. `contentToParasWithImages` 函数处理文本，在占位符位置插入图片
3. 未匹配的图片会被收集到 `unmatchedImgs` 中
4. 在文档末尾添加 "附录：其他界面截图" 章节，将所有未匹配图片堆积在这里

**问题**：即使图片与功能模块相关，如果匹配算法没有完全成功，图片就会被堆积到末尾，而不是与对应的功能文字放在一起。

**目标**：改进图片匹配逻辑，确保图片能够尽可能与对应的功能文字放在一起，避免全部堆积在末尾。

---

## 具体修改方案

### 修改1：改进 `proxy.js` 中的表格生成

**文件**：`proxy.js`

**位置**：需要在 `buildDesignDocx` 函数中添加或修改表格生成逻辑。

当前代码使用 `docx` 库的 `Table` 组件生成表格，但问题可能在于如何将 markdown 表格转换为 docx 表格。

实际上，查看 `buildDesignDocx` 函数（第 311 行开始），我发现它已经有了一些表格生成逻辑（第 325-334 行的人员分工表，第 336-347 行的功能目标表）。

问题可能在于：AI 生成的内容中包含 markdown 格式的表格，而不是 docx 表格。

**解决方案**：
1. 在 `contentToParasWithImages` 函数中添加对 markdown 表格的识别和转换
2. 将 markdown 表格转换为 docx 表格对象

具体修改：
- 在 `contentToParasWithImages` 函数中，识别 markdown 表格（以 `|` 开头的行）
- 解析表格内容，生成 docx 的 `Table` 对象
- 将表格对象添加到 `result` 数组中

### 修改2：改进图片匹配逻辑

**文件**：`proxy.js`

**位置**：`matchImagesToPlaceholders` 函数

当前逻辑：
1. 首先匹配 `[ARCHITECTURE_IMAGE]` 占位符
2. 然后匹配 `[WIREFRAME]` 占位符，先通过标题相似度匹配，再通过顺序匹配

问题：
1. 如果图片数量多于占位符数量，多余的图片会被放入 `unmatchedImgs`
2. 这些未匹配的图片会被堆积到文档末尾

改进方案：
1. 减少使用占位符的方式，改为直接在功能模块描述后插入图片
2. 或者，改进匹配算法，确保每个功能模块都能有一个对应的图片
3. 如果图片数量少于功能模块数量，可以考虑重复使用图片，或者让某些模块没有图片
4. 如果图片数量多于功能模块数量，可以考虑将多余的图片作为补充材料，但不应该全部堆积在末尾

具体修改：
- 在 `contentToParasWithImages` 函数中，当遇到功能模块标题时，尝试从 `images` 中找到对应的图片
- 如果找到匹配的图片，立即在该位置插入图片
- 而不是等到最后才统一处理未匹配的图片

### 修改3：调整 AI 提示词

**文件**：`秒著.html`

**位置**：`genDesignSpec` 函数中的 prompt

当前的 prompt 已经比较详细，但需要明确以下几点：
1. 明确要求使用表格格式而不是 markdown 格式（虽然我们会通过后处理转换，但最好让 AI 生成更清晰的内容）
2. 明确要求图片和功能文字放在一起

## 实施步骤

### 步骤1：修改 `proxy.js` - 添加 markdown 表格转 docx 表格功能

**修改位置**：`contentToParasWithImages` 函数

**新增函数**：`parseMarkdownTable` 和 `createDocxTable`

```javascript
// 解析 markdown 表格
function parseMarkdownTable(lines, startIndex) {
  const tableLines = [];
  let i = startIndex;
  while (i < lines.length && lines[i].trim().startsWith('|')) {
    tableLines.push(lines[i]);
    i++;
  }
  
  if (tableLines.length < 2) return null;
  
  // 解析表头
  const headers = tableLines[0].split('|').map(h => h.trim()).filter(h => h);
  // 跳过分隔行（包含 --- 的行）
  const dataRows = [];
  for (let j = 2; j < tableLines.length; j++) {
    const cells = tableLines[j].split('|').map(c => c.trim()).filter(c => c);
    if (cells.length > 0) {
      dataRows.push(cells);
    }
  }
  
  return { headers, dataRows, endIndex: i };
}

// 创建 docx 表格
function createDocxTable(headers, dataRows) {
  const { Table, TableRow, TableCell, Paragraph, TextRun, WidthType, AlignmentType } = require('docx');
  
  const colCount = headers.length;
  const colWidth = Math.floor(10000 / colCount);
  
  // 创建表头行
  const headerRow = new TableRow({
    children: headers.map(h => new TableCell({
      borders: makeBorder(),
      width: { size: colWidth, type: WidthType.DXA },
      shading: { fill: 'DDEEFF', type: ShadingType.CLEAR },
      margins: { top: 60, bottom: 60, left: 100, right: 100 },
      children: [new Paragraph({ 
        alignment: AlignmentType.CENTER,
        children: [new TextRun({ text: h, bold: true, size: 22, font: '宋体' })] 
      })]
    }))
  });
  
  // 创建数据行
  const dataRowsObj = dataRows.map(row => new TableRow({
    children: row.map(cell => new TableCell({
      borders: makeBorder(),
      width: { size: colWidth, type: WidthType.DXA },
      margins: { top: 60, bottom: 60, left: 100, right: 100 },
      children: [new Paragraph({ 
        children: [new TextRun({ text: cell, size: 22, font: '宋体' })] 
      })]
    }))
  }));
  
  return new Table({
    width: { size: 10000, type: WidthType.DXA },
    rows: [headerRow, ...dataRowsObj]
  });
}
```

然后在 `contentToParasWithImages` 函数中，当识别到 markdown 表格时，调用这些函数：

```javascript
// 在 contentToParasWithImages 函数中

    const tableData = parseMarkdownTable(lines, i);
    if (tableData) {
      const table = createDocxTable(tableData.headers, tableData.dataRows);
      result.push(table);
      i = tableData.endIndex - 1; // 跳过已处理的行
      continue;
    }
  }
  
  // ... 其余代码保持不变
}
```

### 步骤2：修改 `proxy.js` - 改进图片匹配和插入逻辑

**修改位置**：`contentToParasWithImages` 函数

主要改进：
1. 在识别到功能模块标题时，主动查找匹配的图片
2. 如果找到匹配的图片，立即在标题后插入图片
3. 对于未匹配的图片，不再全部堆积到末尾，而是尝试在相关段落中插入

```javascript
// 新增：根据标题查找匹配的图片
function findMatchingImage(title, images, usedIndices) {
  for (let i = 0; i < images.length; i++) {
    if (usedIndices.has(i)) continue;
    const img = images[i];
    if (!img.section) continue;
    
    const similarity = textSimilarity(title, img.section);
    if (similarity > 0.6) {
      return { index: i, image: img, similarity };
    }
  }
  return null;
}

// 修改 contentToParasWithImages 函数
function contentToParasWithImages(text, images, archImg) {
  const lines = (text || '').split('\n');
  const { mapping, unmatchedImgs } = matchImagesToPlaceholders(lines, images, archImg);
  const result = [];
  let skipTocSection = false;
  const usedImageIndices = new Set(); // 追踪已使用的图片
  
  // 记录已使用的图片索引
  Object.values(mapping).forEach(item => {
    const idx = images.findIndex(img => img.name === item.img.name && img.from === item.img.from);
    if (idx >= 0) usedImageIndices.add(idx);
  });
  
  for (let i = 0; i < lines.length; i++) {
    const trimmed = lines[i].trim();
    
    // 处理占位符（原有逻辑）
    if (/^\[WIREFRAME(?::.+)?\]$/.test(trimmed) || trimmed === '[ARCHITECTURE_IMAGE]') {
      if (mapping[i]) {
        result.push(...makeImagePara(mapping[i].img, mapping[i].caption));
        const idx = images.findIndex(img => img.name === mapping[i].img.name && img.from === mapping[i].img.from);
        if (idx >= 0) usedImageIndices.add(idx);
      }
      continue;
    }
    
    // 新增：识别功能模块标题，尝试匹配图片
    // 匹配模式：### 5.1 功能模块名称 或 ### 功能模块名称
    const moduleMatch = trimmed.match(/^###\s+(?:\d+\.\d+\s+)?(.+)$/);
    if (moduleMatch) {
      const moduleTitle = moduleMatch[1];
      // 尝试查找匹配的图片
      const match = findMatchingImage(moduleTitle, images, usedImageIndices);
      if (match) {
        // 在功能模块内容后插入图片（稍后处理）
        // 这里我们先记录这个匹配关系
        if (!window.moduleImageMatches) window.moduleImageMatches = {};
        window.moduleImageMatches[moduleTitle] = match;
        usedImageIndices.add(match.index);
      }
    }
    
    // 新增：识别 "wireframe" 关键词，在此处插入已匹配的图片
    if (/^wireframe$/i.test(trimmed) && window.moduleImageMatches) {
      // 查找当前上下文对应的功能模块
      // 向前查找最近的模块标题
      let moduleTitle = null;
      for (let j = i - 1; j >= 0; j--) {
        const prevMatch = lines[j].trim().match(/^###\s+(?:\d+\.\d+\s+)?(.+)$/);
        if (prevMatch) {
          moduleTitle = prevMatch[1];
          break;
        }
      }
      if (moduleTitle && window.moduleImageMatches[moduleTitle]) {
        const match = window.moduleImageMatches[moduleTitle];
        result.push(...makeImagePara(match.image, match.image.section || '界面示意图'));
        delete window.moduleImageMatches[moduleTitle]; // 使用后删除
        continue;
      }
    }
    
    // 其余代码保持不变...
    if (!trimmed) {
      if (!skipTocSection) {
        result.push(new Paragraph({ spacing: { after: 60 }, children: [] }));
      }
      continue;
    }
    
    if (/^#{1,3}\s/.test(trimmed)) {
      const title = cleanMarkdown(trimmed.replace(/^#+\s/, ''));
      const lvl = trimmed.startsWith('###') ? 3 : trimmed.startsWith('##') ? 2 : 1;
      
      if (title.includes('目录')) {
        skipTocSection = true;
        continue;
      }
      if (skipTocSection && lvl === 2) {
        skipTocSection = false;
      }
      if (!skipTocSection) {
        result.push(makeHeading(title, lvl));
      }
    } else if (!skipTocSection) {
      if (/^[-\*•]\s/.test(trimmed)) {
        result.push(new Paragraph({ bullet: { level: 0 }, spacing: { after: 60 }, children: [new TextRun({ text: cleanMarkdown(trimmed.replace(/^[-\*•]\s/, '')), size: 22, font: '宋体' })] }));
      } else if (/^\d+\.\s/.test(trimmed)) {
        result.push(new Paragraph({ numbering: { reference: 'default', level: 0 }, spacing: { after: 60 }, children: [new TextRun({ text: cleanMarkdown(trimmed.replace(/^\d+\.\s/, '')), size: 22, font: '宋体' })] }));
      } else {
        result.push(makePara(cleanMarkdown(trimmed)));
      }
    }
  }
  
  // 处理未匹配的图片：不再全部堆积到末尾，而是尝试在相关位置插入
  // 如果有未匹配的图片，尝试在相关内容后插入
  const trulyUnmatched = images.filter((img, idx) => !usedImageIndices.has(idx));
  if (trulyUnmatched.length > 0) {
    // 不再添加到末尾，而是尝试在找到相关段落后插入
    // 这里简化处理：如果没有匹配到特定位置，就不显示
    // 或者可以添加一个简短的提示
    console.log(`${trulyUnmatched.length} 张图片未匹配到对应位置`);
  }
  
  return { paragraphs: result, unmatchedImgs: [] }; // 返回空数组，不再堆积到末尾
}
```

### 步骤3：修改 `proxy.js` - 添加文本表格生成函数

如果需要在 AI 生成的内容中使用纯文本表格（而不是 docx 表格对象），可以添加以下函数：

```javascript
// 创建文本格式的表格（用于在 AI 生成的纯文本内容中）
function createTextTable(headers, rows) {
  const colWidths = headers.map((h, i) => {
    const headerLen = String(h).length;
    const maxDataLen = Math.max(...rows.map(r => String(r[i] || '').length));
    return Math.max(headerLen, maxDataLen, 6) + 2;
  });
  
  const makeLine = (left, mid, right, fill) => {
    return left + colWidths.map(w => fill.repeat(w)).join(mid) + right;
  };
  
  const makeRow = (cells) => {
    return '│' + cells.map((c, i) => ' ' + String(c || '').padEnd(colWidths[i] - 1, ' ')).join('│') + '│';
  };
  
  const lines = [
    makeLine('┌', '┬', '┐', '─'),
    makeRow(headers),
    makeLine('├', '┼', '┤', '─'),
    ...rows.map(row => makeRow(row)),
    makeLine('└', '┴', '┘', '─')
  ];
  
  return lines.join('\n');
}
```

然后在 AI 提示词中说明："请使用文本表格格式，而不是 markdown 表格格式。"
