## 1. 系统架构概述

### 1.1 整体架构
秒著采用**前端单页应用 + 后端代理**的架构设计，实现了：
- 前端：HTML + 原生JavaScript，零依赖
- 后端：Node.js 代理服务器
- AI：多模型API支持（含免费模型）

### 1.2 目录结构
```
软著生成器/
├── 秒著.html              # 前端主页面
├── proxy.js               # Node.js后端代理
├── package.json           # npm依赖
├── package-lock.json      # 依赖锁定
├── 启动秒著.bat           # Windows启动脚本
├── 启动秒著.command       # macOS启动脚本
├── PRD.md                 # 产品需求文档
└── 技术设计文档.md        # 本文档
```

## 2. 前端设计

### 2.1 技术栈
- **原生JavaScript**：无第三方框架依赖
- **HTML5**：语义化标签
- **CSS3**：自定义样式系统，采用 Fraunces 和 Nunito 字体
- **CDN依赖**：
  - mammoth.js - 文档解析
  - pdf.js - PDF解析
  - JSZip - ZIP文件处理

### 2.2 页面结构
页面分为4个步骤模块，通过 `currentStep` 状态控制显示：

| Step | 功能 |
|------|------|
| 1 | 配置API（选择API提供商 + 输入API Key） |
| 2 | 上传文件或粘贴文本 |
| 3 | 确认信息（AI提取 + 手动编辑） |
| 4 | 生成结果（文档预览 + 下载） |

> **v1.5更新**：新增OpenRouter免费模型，无需配置API Key即可使用。新增流式输出功能，实时预览生成内容。

### 2.3 关键功能实现

## 5. 关键算法

### 5.1 文本相似度计算
```javascript
textSimilarity(a, b)
  1. 预处理：去除空白、数字、标点
  2. 完全匹配：返回 1.0
  3. 包含关系：返回 0.8
  4. 字符交集：返回 common / max_length
```
