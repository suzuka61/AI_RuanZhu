## 1. 系统架构概述

### 1.1 整体架构
秒著采用**前端单页应用 + 后端代理**的架构设计，实现了：
- 前端：HTML + 原生JavaScript，零依赖
- 后端：Node.js 代理服务器
- AI：火山引擎API

### 1.2 目录结构
```
软著生成器/
├── 秒著.html              # 前端主页面
├── proxy.js               # Node.js后端代理
├── package.json           # npm依赖
├── package-lock.json      # 依赖锁定
├── 启动秒著.bat           # Windows启动脚本
├── 启动秒著.command       # macOS启动脚本
├── PRD.md                 # 产品需求文档
└── 技术设计文档.md        # 本文档
```

## 2. 前端设计

### 2.1 技术栈
- **原生JavaScript**：无第三方框架依赖
- **HTML5**：语义化标签
- **CSS3**：自定义样式系统，采用 Fraunces 和 Nunito 字体
- **CDN依赖**：
  - mammoth.js - 文档解析
  - pdf.js - PDF解析
  - JSZip - ZIP文件处理

### 2.2 页面结构
页面分为4个步骤模块，通过 `currentStep` 状态控制显示：

| Step | 功能 |
|------|------|
| 1 | 配置API（选择API提供商 + 输入API Key） |
| 2 | 上传文件或粘贴文本 |
| 3 | 确认信息（AI提取 + 手动编辑） |
| 4 | 生成结果（文档预览 + 下载） |

> **v1.4更新**：简化模型配置，移除手动模型选择。用户只需选择API提供商（火山、DeepSeek、智谱、Kimi、MiniMax），系统自动使用对应提供商的默认模型。降低了用户配置复杂度。

### 2.3 关键功能实现

#### 2.3.1 文档解析
```javascript
// DOCX 解析流程
1. 使用 mammoth.js 提取文本
2. 使用 JSZip 解压 docx
3. 遍历 word/document.xml 提取图片
4. 解析图片所属章节标题

// PDF 解析流程
1. 使用 pdf.js 加载PDF
2. 逐页提取文本内容
```

#### 2.3.2 AI调用与API提供商配置

#### 支持的API提供商

| 提供商 | 标识 | 默认模型 | Base URL |

| 火山引擎方舟 | volcengine | doubao-seed-2-0-code | ark.cn-beijing.volces.com |
| DeepSeek | deepseek | deepseek-chat | api.deepseek.com |
| 智谱 GLM | zhipu | glm-4.7-thinking | open.bigmodel.cn |

| MiniMax | minimax | MiniMax-Text-01 | api.minimax.chat |

#### 前端配置逻辑
```javascript
// 提供商默认模型映射
const PROVIDER_DEFAULT_MODELS = {
  volcengine: 'doubao-seed-2-0-code',
  deepseek: 'deepseek-chat',
  zhipu: 'glm-4.7-thinking',
  moonshot: 'moonshot-v1-8k',
  minimax: 'MiniMax-Text-01'
};

// 用户选择提供商后自动设置默认模型
function updateModelForProvider() {
  const provider = document.getElementById('providerSelect').value;
  config.provider = provider;
  config.model = PROVIDER_DEFAULT_MODELS[provider] || 'doubao-seed-2-0-code';
}
```

## 3. 后端设计

### 3.1 技术栈
- **Node.js**：原生 http 模块
- **docx npm包**：v7.1.0

| 路径 | 方法 | 功能 |
|------|------|------|
| / | GET | 返回 秒著.html |

## 5. 关键算法

### 5.1 文本相似度计算
```javascript
textSimilarity(a, b)
  1. 预处理：去除空白、数字、标点
  2. 完全匹配：返回 1.0
  3. 包含关系：返回 0.8
  4. 字符交集：返回 common / max_length
```

### 5.2 Markdown表格解析
```javascript
状态机逐行解析：
  检测表格起始 → 收集表格行 → 识别分隔线 → 识别标题 → 生成表格
```
